# Skill: Settings and configuration

## Purpose

Use this skill when working on project settings, feature toggles, module activation, admin configuration flows, or platform-level project configuration.

---

## Objective

Turn the starter into a configurable product base.

---

## Areas to support

- project information
- branding basics
- enabled modules
- billing activation state
- customer portal options
- setup completeness signals
- future configuration entry points

---

## Rules

- think from an admin UX perspective
- keep settings typed and validated
- make module activation explicit
- preserve clean separation between secure backend handling and dashboard UX
- keep the settings model extensible but understandable
- avoid hidden implicit configuration

---

## Expected outcomes

- understandable settings UI
- safe update flows
- controlled module behavior
- a starter that behaves like a configurable product
