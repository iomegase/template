# Skill: Dashboard with shadcn/ui

## Purpose

Use this skill when building or refactoring the dashboard UI based on shadcn/ui, including layout shell, navigation, cards, tables, settings pages, and dashboard-specific user flows.

---

## Objective

Create a clean, modern, production-ready dashboard using:

- shadcn/ui primitives
- shadcn/ui blocks where useful
- Tailwind CSS
- consistent spacing and visual hierarchy
- reusable layout patterns

---

## Baseline

The project starts from the official dashboard starter block.
The goal is not to discard that baseline, but to turn it into a reusable starter shell.

---

## Rules

- preserve useful imported starter patterns
- extract reusable layout pieces when justified
- keep sidebar and header logic maintainable
- use consistent cards, actions, tables, dialogs, and forms
- keep the dashboard easy to adapt for multiple client projects
- avoid decorative complexity that hurts reuse

---

## Recommended dashboard sections

- public authentication entry point
- admin dashboard
- super-admin dashboard
- customer dashboard
- settings area
- billing area
- CRUD list/form/detail areas

---

## Expected outcomes

- reusable dashboard shell
- coherent navigation
- consistent UX patterns
- maintainable shadcn/ui integration
