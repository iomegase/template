# Skill: Super-admin platform layer

## Purpose

Use this skill when implementing the platform owner area and the productization layer of the starter.

---

## Objective

Allow the platform owner to manage projects, admins, and module activation from a dedicated super-admin space.

---

## Scope

- super-admin route area
- project registry
- project state
- admin management
- module toggles
- setup completeness visibility
- clone-readiness visibility

---

## Rules

- keep the first version lean
- only model project/platform complexity that is actually useful
- preserve starter simplicity
- preserve clone-friendliness
- avoid unnecessary multi-tenant overengineering
- focus on real operational controls

---

## Expected outcomes

- useful operational back-office for the platform owner
- scalable base for future client onboarding
- better productization of the starter
